Student handbook

n8n — Campus Tech Fest Registration System

Five workflows, one running story: from your first click to an AI agent that answers real student questions. Work through them in order — each one builds on the last. Check off steps as you go; your progress is saved in this browser.

Before you begin

Setup

Everything below assumes you have your own n8n instance and the 5 workflow files.

How to import a workflow

  1. Open a blank workflow in n8n.
  2. Click the ⋯ menu (top right, next to Publish).
  3. Choose Import from File and select the .json file.
  4. Click Execute workflow to test it manually, or Listen for Test Event if it starts with a Webhook.
Workflow Files

Copy & Paste the Workflows

The complete JSON for Workflows 1–4 is right here — no separate files needed. Two ways to use each one:

  1. Paste onto canvas (easiest) — click Copy JSON, open a workflow in n8n, click anywhere on the canvas, and press Ctrl+V (Cmd+V on Mac). The nodes appear instantly.
  2. Save as a file — paste into a text editor, save as workflow.json, then in n8n use ⋯ → Import from File.
Heads up: the OpenRouter node in Workflow 4 imports with no credential selected — open it and choose your OpenRouter API Key credential under Authentication before running.

Foundations

How n8n Thinks

Three ideas carry the entire handbook. Everything else is just new nodes plugged into this same shape.

TermMeaning
TriggerWhat starts a workflow — a click, a webhook call, a schedule.
NodeOne step — transform data, call an API, branch, respond.
ConnectionThe line between nodes. Data flows along it as JSON, left to right.

Expressions

Anything inside {{ }} is evaluated live using the current item's data.

{{ $json.fieldName }}              // a field on the current item
{{ $json.body.name }}              // webhook payloads land under .body
{{ $('Node Name').item.json.x }}   // reach back to an earlier node
{{ $now.toISO() }}                 // current timestamp

Workflow 2 · Basic

Webhook Registration

A student POSTs their name to a URL; the workflow generates a registration ID, composes a message, and replies with JSON — all live, no polling.

Webhook→ Generate ID→ Build Confirmation→ Respond
NodeTypeConfig
WebhookWebhookPOST, path fest-register, Response Mode = Using Respond to Webhook Node.
Generate Registration IDSetregistrationId built from today's date + a random number.
Build ConfirmationSetconfirmationMessage — reaches back to the Webhook node's data with $('Webhook').item.json.body.name.
Respond to WebhookRespond to WebhookReturns { registrationId, message, registeredAt }.

Your checklist

curl -X POST https://<your-n8n>/webhook-test/fest-register \
  -H "Content-Type: application/json" \
  -d '{"name": "Asha"}'
Expected response{ "registrationId": "TF-...", "message": "Hi Asha, you're registered...", "registeredAt": "..." }

Common pitfalls

Data path

Forgetting .body — it's $json.body.name, not $json.name.

Config

Response Mode not set to "Using Respond to Webhook Node" — the last node won't be able to control the reply.

URL

Confusing the Test URL (only live while listening) with the Production URL.

Workflow 5 · Your Turn — no starter file provided

The Personalized Fest Assistant

No starter file this time — build it from a blank canvas. But look closely: every node type here is one you've already used in Workflows 1–4. This is really a test of whether those four stuck, not a fifth new concept.

Requirements

  1. Trigger — a webhook (POST), expecting { "name": "...", "branch": "...", "year": N, "question": "..." }.
  2. Build a student profile — a Set node shaping the incoming fields into a clean record (exactly what Workflow 1's Set node did, just with real input).
  3. Personalize the AI call — same OpenRouter endpoint and credential as Workflow 4, but the system prompt is built dynamically from the profile's name/branch/year.
  4. Extract and respond — return { "greeting": "Hi Priya!", "answer": "..." }.

Where to look

How to check your work

Send a test request, then send a second one changing only branch or year. Confirm the AI's answer actually shifts tone — that's proof the personalization is really wired through, not just decoration on the response.

Stretch goal: add a default value for any missing field (e.g. if branch isn't provided, fall back to "a student") using an expression instead of requiring every field.
