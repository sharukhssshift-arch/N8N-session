Student handbook

n8n — Campus Tech Fest Registration System

Eight workflows, one running story: from your first click to an AI agent that answers real student questions, logs feedback to a spreadsheet, pulls in live news, and guards against duplicate signups. Work through them in order — each one builds on the last. Check off steps as you go; your progress is saved in this browser.

Before you begin

Setup

Everything below assumes you have your own n8n instance and the 8 workflow files.

How to import a workflow

  1. Open a blank workflow in n8n.
  2. Click the ⋯ menu (top right, next to Publish).
  3. Choose Import from File and select the .json file.
  4. Click Execute workflow to test it manually, or Open Chat if it starts with a Chat Trigger.
Workflow Files

Copy & Paste the Workflows

The complete JSON for Workflows 1–4, 6, 7, and 8 is right here — no separate files needed. Two ways to use each one:

  1. Paste onto canvas (easiest) — click Copy JSON, open a workflow in n8n, click anywhere on the canvas, and press Ctrl+V (Cmd+V on Mac). The nodes appear instantly.
  2. Save as a file — paste into a text editor, save as workflow.json, then in n8n use ⋯ → Import from File.
Heads up: the OpenRouter node in Workflow 4 imports with no credential selected — open it and choose your OpenRouter API Key credential under Authentication before running. The Google Sheets nodes in Workflows 6 and 8 also import without a credential, and their documentId is a placeholder — swap in your own spreadsheet's ID (from its URL) before running.

Foundations

How n8n Thinks

Three ideas carry the entire handbook. Everything else is just new nodes plugged into this same shape.

TermMeaning
TriggerWhat starts a workflow — a click, a webhook call, a schedule.
NodeOne step — transform data, call an API, branch, respond.
ConnectionThe line between nodes. Data flows along it as JSON, left to right.

Expressions

Anything inside {{ }} is evaluated live using the current item's data.

{{ $json.fieldName }}              // a field on the current item
{{ $json.chatInput }}              // the student's typed message, from a Chat Trigger
{{ $('Node Name').item.json.x }}   // reach back to an earlier node
{{ $now.toISO() }}                 // current timestamp

Common pitfalls

Data path

Reaching for $json.name out of habit — with a Chat Trigger it's $json.chatInput.

Field name

Naming the final field anything other than output — the chat panel won't know what to display.

Node reference

Renaming the trigger node breaks any $('When chat message received') reference elsewhere — update both together.

Workflow 5 · Your Turn — no starter file provided

The Personalized Fest Assistant

No starter file this time — build it from a blank canvas. But look closely: every node type here is one you've already used in Workflows 1–4. This is really a test of whether those four stuck, not a fifth new concept.

Requirements

  1. Trigger — a Chat Trigger. The student types one free-text message that can mention anything about themselves alongside their question, e.g. Hi, I'm Priya, a 3rd-year Computer Science student. How do I register for the fest? No JSON, no structured fields.
  2. Capture the message — a Set node that pulls chatInput into a clearly-named field (exactly what Workflow 1's Set node did, just with real input).
  3. Personalize the AI call — same OpenRouter endpoint and credential as Workflow 4, with a system prompt that tells the model to personalize using whatever name/year/branch the student happens to mention — the model does the "extraction," not you.
  4. Extract and reply — pull the answer out of the API response into a field literally named output (same convention as Workflow 4) so it shows up as the chat reply.

Where to look

Four ideas, all recombined from Workflows 1–4 — no new node types:

How to check your work

Open the chat panel and send Hi, I'm Priya, a 3rd-year Computer Science student. How do I register for the fest?, then send a second message changing the year/branch, e.g. I'm Rohan, a 1st-year Mechanical student — how do I register? Confirm the AI's answer actually shifts tone between the two — that's proof the personalization is really wired through, not just decoration on the response.

Stretch goal: try a message that mentions nothing about the student at all — just a bare question — and confirm the assistant still replies sensibly instead of breaking.
